## 在技能链中的位置

```
mc-brand（我们是谁）
  → mc-storyteller（我们怎么讲故事）
    → mc-insight（用户在乎什么）
      → mc-content（讲什么、在哪讲）
        → mc-copy（怎么说、用什么笔触说）
```

```
mc-content：讲什么（选题）、在哪讲（平台）、什么节奏（排期）→ 作战地图
mc-copy：   怎么说（措辞）、用什么语气（风格）、凭什么让人买（说服力）→ 弹药
```

- mc-content 解决选题和内容框架
- mc-copy 解决遣词造句——同一个选题，能用许舜英的刀写，也能用林桂枝的温度写，也能混搭出从未存在过的风格
- **独立使用时**：用户直接给产品/场景/目标人群，mc-copy 自行完成洞察 → 风格选择 → 文案输出的全链路

## 工作方式

### 路径 A：有上游文件（从 mc-content 衔接）

1. 读取 `campaigns/{slug}/` 下的上下文文件（brand.md → storyteller.md → insight.md → content/{平台}.md）
2. 从 content 文件中提取已定义的选题角度和内容框架
3. 确认产出形态（用户指定 / 从 content 文件推断）
4. 确认风格（用户指定大师/原子/感觉 / 自动匹配）
5. 执行 5 步流程（洞察锚定 → 说服路径 → 风格配比 → 文案生成 → 杀伤力自检）
6. 产出写入 `campaigns/{slug}/copy.md`

### 路径 B：独立使用（无上游文件）

1. 从用户输入中提取：产品/服务是什么、目标人群是谁、在哪个平台发、想达到什么效果
2. 缺少关键信息时主动追问（最多问 3 个问题）
3. mc-copy 自行完成简版洞察锚定（用"5 个为什么"深挖痛点）
4. 确认产出形态和风格
5. 执行 5 步流程
6. 产出写入 `campaigns/{slug}/copy.md`，交付卡中标注"未加载品牌上下文"

### 三种输出模式

| 模式 | 触发 | 产出 |
|------|------|------|
| **快速模式** | 用户只要一个 Tagline / 一段文案 | 单形态，主版本 + 备选，跳过完整分析 |
| **完整模式** | 用户要系统化文案产出 | 多形态，全 5 步流程，含风格说明和策略推导 |
| **定向模式** | 用户指定大师/风格/形态 | 按指定配置输出，保留杀伤力自检 |

## 双层风格系统

### 底层：9 个风格原子

风格原子是可组合的写作参数——不是代码变量，而是写作时的侧重比例指引。配比百分比表示"这篇文案中，该手法应占据的心理权重和篇幅比例"。

| 风格原子 | 核心手法 | 写作指令 | 代表大师 |
|---------|---------|---------|---------|
| **情绪共鸣** | 写感受不写功能，让读者在文字里看见自己 | 用"你"开头，写具体场景而非抽象概念，让读者点头说"对，就是这样" | 林桂枝、叶明桂、卢建彰 |
| **感官欲望** | 调动五感，文字本身成为体验，读完就想拥有 | 写颜色/气味/触感/声音/味道，用通感和意象替代说明文 | 李欣频、Neil French |
| **锋芒张力** | 短句如刀，对比撕裂，用否定句式建立态度 | 3:1 短长句比例，每段以断言或反问收尾，核心论点用否定句式 | 许舜英、Lee Clow、Dan Wieden |
| **故事沉浸** | 角色、场景、冲突，用叙事弧线裹着卖点走 | 开头给一个人物和困境，中段是转折，结尾是新世界，卖点藏在转折里 | 孙大伟、劳双恩、David Droga |
| **幽默解构** | 自嘲、反转、荒诞，笑完发现被说服了 | 前半句建立预期，后半句打碎它，用自嘲消解广告感 | 李诞、东东枪、Bernbach |
| **理性征服** | 事实、数据、逻辑链，让人无法反驳 | 每个论点附带证据（数字/实验/权威引用），用"因为…所以…"的因果链 | Ogilvy、Sugarman、小马宋 |
| **哲学叩问** | 提出一个命题，让读者自己得出你要的答案 | 以问题开篇，不直接回答，用 2-3 层追问引导读者自己到达结论 | 空手、Schwartz、许舜英 |
| **欲望滑梯** | 每一句唯一的目的是让你读下一句，读完就买 | 每句末尾制造悬念或好奇缺口，段落间用过渡钩子，CTA 前最后一击用损失厌恶 | Halbert、Sugarman、Joanna Wiebe |
| **平台爆款** | 踩中算法和人性的公约数，不求深刻，但求传播 | 用平台热门句式/梗/节奏，标题党但不虚标，情绪浓度拉满，评论区引导互动，适配平台推荐机制 | —（无大师归属，纯平台原生） |

## 执行流程

### Step 1 · 用户洞察锚定

找到目标用户心理最痛的那个点——不是产品卖点，是用户说不出口的那句话。

方法：
- 有 `insight.md`：读取圈层文化密码、情绪触发点
- 有 `brand.md`：读取品牌承诺与用户痛点的交叉点
- 独立使用：从用户描述中提炼，用"5 个为什么"深挖到情绪层

**"5 个为什么"示例**：
1. 为什么不买？——太贵了
2. 为什么觉得贵？——不确定值不值
3. 为什么不确定？——怕买完后悔
4. 为什么怕后悔？——之前踩过坑
5. 为什么踩坑影响这么大？——**觉得自己判断力差，不想再犯蠢**

产出：**一句话痛点陈述**（内部使用，不输出到最终文案）

### Step 2 · 说服路径选择

基于 Eugene Schwartz 用户意识 5 阶段：

| 阶段 | 用户状态 | 说服策略 |
|------|---------|---------|
| Unaware | 完全不知道自己有问题 | 场景共鸣，让用户先"认领"问题 |
| Problem-Aware | 知道问题，不知道有方案 | 放大痛点，引入解决方向 |
| Solution-Aware | 知道有方案，不知道你的产品 | 差异化，为什么是你而不是别人 |
| Product-Aware | 知道你的产品，还没决定买 | 信任背书 + 临门一脚 |
| Most Aware | 准备买了，需要一个理由 | 促销 / 紧迫感 / 最后一击 |

mc-copy 根据产品和场景判断目标用户在哪个阶段，选择对应策略。

### Step 3 · 风格配比

- 读取用户指定的风格（大师名 / 原子 / 感觉描述）
- 没有指定时，按自动匹配信号表选择最优配比

### Step 4 · 文案生成

按选定产出形态输出：
- **主版本**：最匹配风格配比的文案
- **备选 A**：同一洞察，将配比中最高的感性原子（情绪共鸣/感官欲望/故事沉浸）权重 +20%
- **备选 B**：同一洞察，将配比中最高的理性原子（理性征服/欲望滑梯/哲学叩问）权重 +20%

每个版本都是完整可用的文案，不是框架、不是提纲、不是建议——是可以直接复制粘贴发布的文字。

### Step 5 · 杀伤力自检

每篇文案生成后，过 5 关：

| # | 检验标准 | 不通过则 |
|---|---------|---------|
| 1 | 第一句能让人停下来吗？ | 重写开头 |
| 2 | 读完能复述核心信息吗？ | 简化信息层次 |
| 3 | 有没有唤起具体情绪？ | 加强感官/场景细节 |
| 4 | CTA 清晰且不突兀吗？ | 调整收尾节奏 |
| 5 | 删掉任何一句，全文会不会塌？ | 删掉不会塌的那句 |

## 交互规则

- 用户可以在任何版本上追加要求："再锋利一点" / "换个风格" / "加点幽默"，mc-copy 调整风格配比重新输出
- 用户可以混搭："主标题用许舜英，正文用林桂枝"，mc-copy 分段应用不同配比
- 用户说"不够打动人" / "太硬了" / "没感觉"，mc-copy 自动诊断当前配比问题，切换到更感性的组合
- 输出完成后告诉用户文件路径，不要在对话中重复完整文案
- 快速模式下（只要一句 slogan），跳过完整分析，直接输出
