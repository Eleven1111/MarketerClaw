| 维度 | mc-seo | mc-geo |
|------|--------|--------|

| 见效周期 | 3-6 个月 | 1-3 个月 |

## 模块 1：关键词研究与策略

### 关键词分层框架

**层级 1：品牌词**
- 品牌名 + 品牌名变体（拼写错误、缩写）
- 品牌名 + 产品名
- 品牌名 + 竞品对比（"品牌 A vs 品牌 B"）
- 策略：防守为主，确保品牌词排名第一，拦截竞品抢占意图

**层级 2：品类核心词（行业词）**
- 高搜索量、高竞争度
- 转化意图强（商业意图 / 购买意图）
- 策略：长期耕耘，需要强域名权重和持续内容投入

**层级 3：问题词 / 场景词**
- "XX 怎么选" / "XX 和 XX 的区别" / "最好的 XX"
- 搜索意图清晰，内容可精准匹配
- 策略：内容营销驱动，博客/指南为主要载体

**层级 4：长尾词**
- 低搜索量、低竞争度、高转化率
- 具体场景化（"适合敏感肌的无香料面霜"）
- 策略：产品页和集合页优化，快速出排名

## 模块 2：页面 SEO（On-Page SEO）

### 核心优化清单

**标题标签 (Title Tag)**
- 长度：50-60 字符（Google）/ 30-50 字符（百度）
- 包含核心关键词（尽量靠前）
- 对用户有吸引力，体现差异化价值
- 每个页面唯一，不重复

**Meta 描述 (Meta Description)**
- 长度：120-160 字符
- 包含关键词（关键词会加粗显示在 SERP）
- 有明确的 CTA（号召性用语）
- 对用户有说服力，提高点击率

**标题层级结构 (H1-H6)**
- 每个页面只有一个 H1，包含核心关键词
- H2/H3 覆盖次级关键词和相关话题
- 结构清晰，帮助爬虫理解内容层级

**URL 结构**
- 简短、描述性强（`/skincare/moisturizer-for-sensitive-skin/`）
- 包含目标关键词
- 使用连字符分隔（-），不用下划线（_）
- 避免无意义的参数和数字

**内容优化**
- 首段出现目标关键词
- 语义相关词（LSI Keywords）自然分布
- 内容深度覆盖话题（SkyScraper 原则：比现有排名结果更好）
- 适当的内容长度（以满足用户需求为准，不以字数为目标）

**图片优化**
- Alt 文字描述性且包含关键词（不关键词堆砌）
- 文件名语义化（`sensitive-skin-moisturizer.jpg`）
- 压缩图片，减少加载时间

**内链策略**
- 相关页面之间互相链接
- 锚文字 (anchor text) 含目标关键词
- 孤立页面（orphan pages）接入内链网络
- 将权重从强页面传导给需要提升的页面

## 模块 3：技术 SEO（Technical SEO）

### 核心技术健康指标

**爬取与索引**
- `robots.txt`：确保不误屏蔽重要页面
- `sitemap.xml`：提交给 Google Search Console / 百度搜索资源平台，及时更新
- 规范化标签 (canonical)：解决重复内容问题
- noindex 标签：只用在真正不需要收录的页面
- 爬取预算：避免无价值页面消耗爬取资源（参数页、过滤页）

**Core Web Vitals（Google 页面体验信号）**

| 指标 | 含义 | 优良标准 | 优化方向 |
|------|------|---------|---------|
| LCP（最大内容渲染） | 主要内容加载速度 | ≤ 2.5s | 图片/视频优化、CDN、服务端渲染 |
| FID / INP（交互响应） | 用户点击到响应的延迟 | INP ≤ 200ms | 减少 JS 执行时间、代码分割 |
| CLS（累积布局偏移） | 视觉稳定性 | ≤ 0.1 | 给图片/广告预设尺寸 |

**移动端优化**
- Google 以移动优先索引（Mobile-First Indexing）为默认
- 响应式设计，移动端体验不低于桌面端
- 字体大小、按钮点击区域、视窗设置

**HTTPS 与安全**
- 全站 HTTPS（非 HTTPS 页面在 Google 有排名惩罚）
- 混合内容（Mixed Content）检查

**结构化数据（Schema Markup）**

| Schema 类型 | 适用页面 | 可获得的富媒体摘要 |
|------------|---------|-----------------|
| `Product` | 产品页 | 价格、评分、库存显示在 SERP |
| `Review` / `AggregateRating` | 评价页 | 星级显示在搜索结果 |
| `FAQPage` | FAQ 页 | 问答直接展开在搜索结果 |
| `HowTo` | 教程页 | 步骤列表显示在搜索结果 |
| `BreadcrumbList` | 全站 | 面包屑导航显示在 URL 下方 |
| `Organization` | 关于我们 | 知识图谱信息 |
| `Article` | 博客 | 发布/更新时间显示 |

**站点结构**
- 扁平化结构（重要页面距首页点击深度 ≤ 3）
- 分页处理（`rel="next"` / `rel="prev"` 或无限滚动的 SEO 处理）
- 重定向链路（避免重定向链/循环重定向）
- 404 页面处理（返回真正的 404 状态码，提供有帮助的内容）

## 模块 4：外链策略（Off-Page SEO）

### 外链质量判断标准

**高价值外链特征：**
- 来自高权重、高相关性的网站
- 自然锚文字（品牌词、URL、自然语句）
- 编辑性链接（非付费购买，内容认可）
- 链接位于正文内容中（非页脚/侧边栏）

**低价值/有害外链特征：**
- 来自垃圾网站、链接农场
- 过度优化的精确匹配锚文字
- 隐藏链接、重定向欺骗
- 大量低质量目录站

### 外链获取策略

**1. 数字 PR（Digital PR）**
最高效的白帽外链策略：
- 发布原创研究/调研报告 → 媒体引用时自然获得链接
- 提供行业专家观点给记者（HARO / Qwoted / 国内媒体库）
- 制作可视化信息图，供其他网站转载并引用来源
- 与行业报告机构合作，成为数据来源

**2. 内容营销外链**
- 客座博客（Guest Post）：在行业媒体/权威博客发布原创内容
- 资源页外链：找到"XX 行业最佳资源"类页面，申请收录
- 损坏链接建设（Broken Link Building）：找到竞品/行业失效链接，提供自己的替代内容

**3. 品牌提及转化**
- 监控未附链接的品牌提及（Unlinked Mentions）
- 联系作者要求添加链接（成功率较高，对方已认可品牌）

**4. 合作与联名**
- 供应商/合作伙伴互链
- 行业协会成员页
- 联名活动、跨品牌合作

**5. 社区与论坛（辅助）**
- Reddit、Quora、知乎等平台的高质量回答（通常 nofollow，但带来流量和品牌曝光）
- 行业论坛参与

## 模块 5：内容 SEO 策略

### Topic Cluster 架构

**Pillar Page（支柱页）**
- 覆盖某一大话题的权威综合页（如"护肤完全指南"）
- 内容全面，长度通常 3000-8000 字
- 内链指向所有相关 Cluster 页

**Cluster Content（集群内容）**
- 每个 Cluster 覆盖支柱页的一个子话题（如"敏感肌护肤怎么选产品"）
- 深度聚焦，1000-2000 字
- 全部回链至 Pillar Page

**好处：**
- 提升主题权威性（Topical Authority）
- 内链结构清晰，帮助爬虫理解内容层级
- 避免关键词蚕食（Keyword Cannibalization）

### 内容发布节奏

- **新站**：高频发布，快速建立内容体量和爬取频率
- **成熟站**：质量优于数量，更新老内容 > 新发低质内容
- **内容更新**：定期审查排名下滑内容，刷新数据和案例

## 模块 6：电商 SEO

### Amazon SEO（A10 算法）

**核心排名因素：**

| 因素 | 权重 | 优化方向 |
|------|------|---------|
| 销售转化率 | 极高 | 优化 listing、价格、评论 |
| 关键词相关性 | 高 | 标题、Bullet Points、后台关键词 |
| 点击率 (CTR) | 高 | 主图质量、价格竞争力 |
| 外部流量 | 中 | 品牌网站、社媒引流至 Amazon |
| 评论数量与评分 | 高 | 售后服务、Vine 计划 |
| 库存健康 | 中 | 避免断货导致排名下降 |

**Listing 优化框架：**

```markdown
## Amazon Listing 优化

### 标题（Title）
- 格式：品牌名 + 产品名 + 核心属性 + 规格 + 适用场景
- 长度：移动端前 80 字符最关键
- 包含前 2-3 个核心关键词

### Bullet Points（5 条）
- 第 1 条：最重要的差异化卖点
- 第 2 条：核心功效/成分
- 第 3 条：适用场景/人群
- 第 4 条：规格/使用方法
- 第 5 条：品牌背书/保障

### 后台关键词（Search Terms）
- 250 字节上限（美国站）
- 不重复标题中已有关键词
- 包含竞品词、替代叫法、西班牙语关键词（美国市场）

### A+ Content / Brand Story
- 对话题权威性和外部流量有辅助作用
- 视觉化呈现差异化卖点

### 评论策略
- Vine 计划（新品期获取真实评论）
- Request a Review 功能（买家确认收货后触发）
- 避免黑帽刷评
```

### 独立站 SEO（Shopify / 自建站）

**独立站特有 SEO 挑战：**
- 产品页重复内容（同款不同颜色/规格）→ canonical 标签处理
- 集合页（Collection）分页 SEO
- 用户生成内容（UGC）处理（评论页索引策略）
- 多货币/多语言站点的 hreflang 配置

**高价值页面优先级：**

1. 首页（品牌核心词）
2. 核心集合页（品类词）
3. 高流量/高转化产品页（长尾词）
4. 博客/指南（信息词、问题词）

## 模块 7：国际化 SEO

### Google 国际化 SEO

**技术配置：**
- `hreflang` 标签：告诉 Google 每个页面针对的语言和地区
- ccTLD（`example.co.uk`）vs 子域名（`uk.example.com`）vs 子目录（`example.com/uk/`）
- 建议优先选子目录（复用主域名权重）

**内容本地化（不只是翻译）：**
- 关键词按目标语言重新研究（直译关键词≠本地用户实际搜索词）
- 货币、日期格式、计量单位本地化
- 文化适应性（避免在本地市场有负面联想的表达）

**目标市场 SEO 重点：**

| 市场 | 主搜索引擎 | 特别注意 |
|------|---------|---------|
| 美国 | Google (92%) | Core Web Vitals、E-E-A-T 信号 |
| 英国 | Google (95%) | 英式英语关键词差异 |
| 德国 | Google (95%) | 隐私政策合规（GDPR）、本地托管偏好 |
| 日本 | Google (77%) | 日语关键词研究（和英语搜索行为差异大） |
| 东南亚 | Google 为主 | 移动端优先，低带宽优化 |
| 中国 | 百度 | 单独技术栈，见下节 |

### 百度 SEO

**与 Google SEO 的关键差异：**

| 维度 | Google | 百度 |
|------|--------|------|
| 爬取 | Googlebot | 百度蜘蛛（Baiduspider） |
| 收录速度 | 快（主动 ping） | 慢（需要主动提交） |
| HTTPS | 强烈偏好 | 支持但非决定性 |
| 反向链接 | 极重要 | 重要，但中文链接生态不同 |
| 页面速度 | Core Web Vitals | 速度重要，但标准不同 |
| 内容质量 | E-E-A-T | 原创性、时效性 |
| JS 渲染 | 支持较好 | 支持较差，建议 SSR |

**百度 SEO 操作清单：**
- 在百度搜索资源平台（ziyuan.baidu.com）提交网站和 sitemap
- 使用百度统计（替代 Google Analytics）
- 百度指数做关键词研究（替代 Google Keyword Planner）
- 百度知道、百度百科、知乎、微信公众号内容与网站形成生态联动
- ICP 备案（面向中国用户的网站必须）
- 服务器部署在中国大陆或香港（影响收录速度）
- 避免使用 Google Fonts、Google Analytics（无法在中国加载）

## 交互规则

- SEO 可以独立于 campaign 使用（"帮我做一个 SEO 审计"）
- 如果有 brief.md 或 strategy.md，关键词策略会自动聚焦到 campaign 相关的品类和市场
- 用户可以提供 Google Search Console 数据或具体排名情况，系统基于实际数据给建议
- 产出的执行清单可以直接交给 SEO 工程师或内容团队落地
- 输出完成后告诉用户文件路径，不要在对话中重复全文
