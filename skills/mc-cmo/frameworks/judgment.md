# 判断框架

mc-cmo 在 🟡 Light Path 和 🔴 Deep Path 中使用此框架做出判断。🟢 Fast Path 不加载此文件。

---

## 快速检查清单（Light Path 使用）

🟡 轻判断路径执行以下 3 项检查，任一项触发则升级为 🔴 深度介入：

### 1. 品牌定位存在？
- 检查当前 campaign 的 `brand.md` 或 `memory/` 下的 `brand-memory.md`
- **存在** → 通过
- **不存在 + 用户请求的是内容/渠道/投放类技能** → 升级为 🔴
- **不存在 + 用户请求的是品牌/调研/竞品类技能** → 通过（这些技能本身就是在建立基础）

### 2. 风险等级？
- **低风险**（纯分析、信息查询、竞品监控）→ 通过
- **中风险**（内容创作、渠道选择）→ 通过，但附带 CMO 建议
- **高风险**（预算分配、投放策略、品牌定位变更）→ 升级为 🔴

风险分类：
| 低风险 | 中风险 | 高风险 |
|--------|--------|--------|
| mc-compete | mc-content | mc-campaign（含预算） |
| mc-monitor | mc-copy | mc-automation（触发型工作流） |
| mc-research | mc-aigc | 品牌定位变更 |
| mc-insight | mc-kol | 全链路编排 |
| mc-report | mc-seo / mc-geo | 预算 > 10万 的投放 |
| mc-review | mc-community | |
| | mc-livestream | |
| | mc-storyteller | |
| | mc-dtc | |

### 3. 信息完整？
- 用户的请求是否包含执行技能所需的最低信息？
- **完整** → 通过
- **缺少关键信息** → mc-cmo 用 CMO 语气追问（不是机械式表单）

## 完整判断流程（Deep Path 使用）

🔴 深度介入按以下 4 步执行：

### Step A: 意图拆解
- 用户真正想要什么？（表面需求 vs 深层需求）
- 这是一次性任务还是系列工作的一部分？
- 是否有隐含的上下文没说出来？

### Step B: 前置条件检查
- 品牌定位是否到位？
- 目标人群是否清晰？
- 是否有足够的数据/调研支撑接下来的决策？
- 是否有竞品分析？
- 渠道选择是否有依据？

缺失的前置条件按重要性排序，向用户建议补全路径。

### Step C: 风险评估
- 这个决策的可逆性如何？（文案可以改，品牌定位改了影响全局）
- 涉及的资源投入量？（写篇文案 vs 启动全链路 campaign）
- 失败的代价？（一篇文案没效果 vs 10 万预算打水漂）

### Step D: 策略建议
基于 A/B/C 的分析，给出：
1. **推荐路径**：应该先做什么、再做什么、最后做什么
2. **理由**：为什么这个顺序最优
3. **风险提示**：如果跳过某些步骤，可能的后果
4. **预期产出**：每个步骤会产出什么，对后续有什么帮助

## cmo_context 输出格式

判断完成后，mc-cmo 向 mc-dispatch 传递的上下文格式：

<!-- CMO Context -->
**判断**：{一句话描述判断结论}
**建议**：{给技能的执行建议，1-2 句话}
**用户阶段**：{new/building/partner} — {该阶段的沟通要求}
<!-- /CMO Context -->

轻判断路径的 cmo_context 尽量简短（1-2 行），深度介入的可以更详细（3-5 行）。
