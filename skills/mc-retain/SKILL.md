- "客户分群" / "用户分层" / "RFM" / "LTV"
- "流失客户" / "客户召回" / "客户快流失了"
- "怎么让老客多买" / "老客贡献太低"
- "订阅流失" / "退订率高" / "续费率低"

## 与其他技能的关系

| | mc-retain | mc-community | mc-automation |
|---|---|---|---|
| 关注点 | **谁**值得留 + **为什么**流失 + **怎么**提客单 | **在哪里**运营社群 + 社群**氛围** | **什么时候**发什么 + 自动化**工作流** |
| 核心输入 | 客户行为数据 + 交易数据 | 社群平台选择 + 品牌人格 | 业务场景 + 触发事件 |
| 核心产出 | 客户分群规则 + 留存/AOV 策略 | 社群架构 + 运营 SOP | 自动化蓝图 + 工作流规格 |
| 协作 | mc-retain 定义分群 → mc-automation 按分群设计触达流 → mc-community 在社群中落地 | 社群数据反馈 → mc-retain 更新分群 | 按 mc-retain 的分群和生命周期设计自动化序列 |

mc-retain 回答"谁值得留、谁快流失、怎么提客单"；mc-community 回答"社群怎么运营"；mc-automation 回答"自动化流程怎么设计"。

## 工作方式

1. 确认留存分析范围：
   - **业务模型**：电商复购 / SaaS 续费 / 订阅制 / 会员制
   - **数据可用性**：有 CRM 数据 / 只有粗略数字 / 完全没有数据
   - **当前痛点**：复购率低 / 客单价低 / 流失严重 / 没有分群
2. 读取 `brand.md`（品牌溢价力影响定价策略）
3. 读取 `analytics.md`（如有历史数据诊断）
4. 产出写入 `campaigns/{project-slug}/retain.md`

## 模块 1：客户分群

### RFM + LTV 分群模型

```markdown
## 客户分群

### 分群维度

| 维度 | 定义 | 数据来源 |
|------|------|---------|
| R (Recency) | 最近一次购买距今天数 | 订单记录 |
| F (Frequency) | 累计购买次数 | 订单记录 |
| M (Monetary) | 累计消费金额 | 订单记录 |
| LTV 预测 | 未来 12 个月预期贡献 | R×F×M 加权 + 品类系数 |
```

### 11 段精细分群

| 分群 | RFM 特征 | 占比参考 | 核心策略 | 运营优先级 |
|------|---------|---------|---------|-----------|
| 💎 至尊客户 | R↑ F↑ M↑ | 2-5% | 专属服务 + VIP 权益 + 新品首发 | ⭐⭐⭐⭐⭐ |
| 🏆 忠诚客户 | R↑ F↑ M→ | 5-10% | 提升客单（交叉销售 + 升级）| ⭐⭐⭐⭐⭐ |
| 🔥 潜力客户 | R↑ F→ M↑ | 5-10% | 提升频次（复购激励 + 定期触达）| ⭐⭐⭐⭐ |
| 🌱 新客 | R↑ F↓ M→ | 15-25% | 快速激活二购（欢迎序列 + 首购后跟进）| ⭐⭐⭐⭐ |
| 🤝 有前途新客 | R↑ F↓ M↑ | 3-5% | 高客单新客重点维护 | ⭐⭐⭐⭐ |
| 😴 即将沉默 | R→ F→ M→ | 10-15% | 唤醒触达（限时优惠 + 个性化推荐）| ⭐⭐⭐ |
| 💤 沉默客户 | R↓ F→ M→ | 10-15% | 召回邮件/短信序列 | ⭐⭐ |
| ⚠️ 高危流失 | R↓ F↑ M↑ | 3-5% | 紧急挽回（曾经的好客户在流失！）| ⭐⭐⭐⭐⭐ |
| 🔻 低价值流失 | R↓ F↓ M↓ | 15-25% | 低成本自动化触达，不投入人工 | ⭐ |
| 🎲 随机客户 | R↓ F↓ M→ | 10-15% | 分析是否为"便宜就买"型 | ⭐ |
| ❄️ 已流失 | R↓↓ F↓ M↓ | 5-10% | 最后一搏 or 放弃 | ⭐ |

### 分群落地规则

```markdown
### CRM/CDP 分群规则

| 分群 | R 条件 | F 条件 | M 条件 |
|------|--------|--------|--------|
| 至尊客户 | 最近 {X} 天内有购买 | 累计 ≥{Y} 次 | 累计 ≥{Z} 元 |
| ...（按品类填入具体阈值）| | | |

⚠️ 以上阈值为建议值，需根据实际数据分布调整。
理想状态：至尊+忠诚+潜力 三群合计贡献 60-70% 收入。
```

### 无数据时的简化分群

用户没有详细数据时，用简化方案：

| 分群 | 判断依据 | 策略 |
|------|---------|------|
| 活跃复购客 | 最近 30 天有购买 + 历史 ≥2 单 | VIP 维护 |
| 新客 | 最近 30 天有购买 + 历史 1 单 | 二购激活 |
| 沉默客 | 30-90 天无购买 | 召回触达 |
| 流失客 | >90 天无购买 | 最后一轮触达 |

## 模块 2：生命周期留存策略

### 六阶段生命周期

```
新客 → 激活 → 成长 → 成熟 → 衰退 → 流失
 │       │       │       │       │       │
 │       │       │       │       │       └─ 最后挽回 or 优雅告别
 │       │       │       │       └─ 召回序列 + 个性化激励
 │       │       │       └─ 交叉销售 + VIP 权益 + 共创参与
 │       │       └─ 复购激励 + 订阅引导 + 社群邀请
 │       └─ 二购跟进 + 使用教育 + 评价邀请
 └─ 欢迎序列 + 首单体验优化 + 品牌故事
```

### 每阶段关键动作

```markdown
## 生命周期策略

### 新客阶段（D0-D7）

**目标：** 让首次购买的体验超出预期，为二购打基础

| 触点 | 时间 | 内容 | 目标 |
|------|------|------|------|
| 订单确认 | D0 | 超预期包装/附赠/手写卡（如适用）| 惊喜感 |
| 欢迎邮件/消息 | D0+2h | 品牌故事 + 使用指南 + 社群邀请 | 建立连接 |
| 使用跟进 | D3-D5 | "用得怎么样？" + 使用技巧 | 培养习惯 |
| 评价邀请 | D7 | 好评引导 + 小激励 | 社会证明 + 参与感 |

### 激活阶段（D7-D30）

**目标：** 推动二购发生

| 触点 | 时间 | 内容 | 目标 |
|------|------|------|------|
| 搭配推荐 | D10 | 基于首购商品的交叉推荐 | 二购动机 |
| 复购激励 | D14 | 限时二购优惠（仅新客可用）| 降低二购门槛 |
| 内容教育 | D21 | 产品深度使用 / 高阶玩法 | 提升感知价值 |
| 即将过期提醒 | D28 | "你的新客权益即将到期" | 紧迫感 |

### 成长阶段（D30-D90）

**目标：** 建立购买习惯，提升 F 和 M

| 策略 | 说明 |
|------|------|
| 订阅引导 | 高频复购品引导订阅（省心+折扣）|
| 社群邀请 | 邀请加入品牌社群/会员群 |
| 等级升级 | 消费达标通知升级 + 权益解锁 |
| 新品预览 | 比普通用户早 48 小时看到新品 |

### 成熟阶段（D90+，活跃复购中）

**目标：** 最大化 LTV，建立情感连接

| 策略 | 说明 |
|------|------|
| VIP 专属 | 专属客服 / 专属折扣 / 生日礼 |
| 共创参与 | 新品内测 / 包装投票 / 产品命名 |
| 推荐奖励 | "带一个朋友，双方各得 XX" |
| 交叉销售 | 基于购买历史的个性化推荐 |

### 衰退阶段（活跃度下降）

**目标：** 尽早发现衰退信号并干预

| 衰退信号 | 干预动作 |
|---------|---------|
| 打开率/点击率连续 3 期下降 | 换内容策略 + 调整触达频率 |
| 购买间隔超过历史均值 1.5 倍 | 个性化召回（基于最近浏览/加购）|
| 开始只买促销款 | 推送非促销的价值内容 |
| 取消订阅/退群 | 发送"我们想改进"调查 |

### 流失阶段

**目标：** 最后一搏或优雅告别

| 策略 | 适用情况 |
|------|---------|
| Win-back 序列（3 封递进）| 沉默 60-90 天 |
| 大额挽回优惠 | 高 LTV 客户，沉默 90-120 天 |
| "我们想念你"调查 | 了解流失原因 |
| 清理列表 | 沉默 >180 天且无任何互动，移出活跃列表 |
```

## 模块 3：客单价提升

### AOV 提升工具箱

```markdown
## 客单价提升策略

### 策略 1：阶梯式包邮

| 当前 AOV | 包邮门槛建议 | 目标 AOV |
|---------|------------|---------|
| ¥{X} | ¥{X × 1.3} | ¥{X × 1.2-1.3} |

**原理：** 包邮门槛设在当前 AOV 的 130% 左右，用户为了"不亏运费"会多加一件。
**注意：** 门槛太高会降低转化率。监控设置后 7 天内的 CVR 变化。

### 策略 2：捆绑销售

| 捆绑模式 | 适用场景 | 定价逻辑 |
|---------|---------|---------|
| 固定套装 | 已验证的搭配组合 | 套装价 = 单品总价 × 85-90% |
| 加购推荐 | "经常一起买" | 加购价 = 单品价 × 70-80% |
| 量贩装 | 高频消耗品 | 单件价随数量递减 |
| 自选套装 | "任选 3 件 ¥XX" | 锚定最高单品价，用户感觉占便宜 |
| 礼盒装 | 送礼场景 | 溢价空间大，重包装和仪式感 |

### 策略 3：赠品激励

| 赠品模式 | 门槛 | 成本控制 |
|---------|------|---------|
| 满额赠品 | 消费满 ¥{X} 赠 {赠品} | 赠品成本 < 增量毛利的 30% |
| 阶梯赠品 | 满 ¥A 赠品 1，满 ¥B 赠品 1+2 | 每个阶梯的赠品价值感递增 |
| 限量赠品 | 前 N 名 / 限时 | 稀缺感提升冲动 |

### 策略 4：升级销售 (Upsell)

| 时机 | 方式 |
|------|------|
| 加购页面 | "升级到 Pro 版本，多 ¥XX" |
| 结账前 | "加 ¥{小金额} 升级到 {更大/更好的版本}" |
| 购后推荐 | "根据你的选择，你可能也需要..." |

### ⚠️ 毛利底线护栏

**所有 AOV 提升策略必须过毛利检查：**

| 检查项 | 底线 |
|--------|------|
| 促销后客单毛利率 | ≥ 正常毛利率的 70% |
| 赠品成本占增量收入比 | ≤ 30% |
| 捆绑折扣不得低于 | 各单品最低可接受售价总和 |
| 包邮补贴上限 | 单笔 ≤ 订单毛利的 15% |

如用户未提供毛利数据，标注 `⚠️ 以下策略未经毛利验证，请在落地前确认利润空间`。
```

## 模块 4：订阅留存（适用于订阅制业务）

### 流失类型区分

| 流失类型 | 特征 | 策略 |
|---------|------|------|
| 价格型流失 | "太贵了" / 只在促销时订阅 | 灵活定价（降档不退订）+ 年付折扣 |
| 价值型流失 | "用不上" / 使用频率低 | 使用教育 + 功能引导 + 个性化推荐 |
| 被动流失 | 支付失败 / 卡过期 | 预到期提醒 + 多次重试 + 备用支付 |
| 竞品型流失 | 转向竞品 | 竞品对比（突出差异化）+ 挽留优惠 |

### 双轨预防

```markdown
## 订阅流失预防

### 轨道 1：使用健康度监控

| 预警信号 | 阈值 | 干预 |
|---------|------|------|
| 登录频率下降 | 低于历史均值 50% | 发送"你可能错过了"邮件 |
| 核心功能不再使用 | 连续 14 天未使用 | 推送使用案例 + 1v1 引导 |
| 仅使用基础功能 | 高级功能使用率 <10% | 功能教育序列 |

### 轨道 2：续费节点干预

| 时间节点 | 动作 |
|---------|------|
| 到期前 30 天 | 价值回顾邮件（"这个月你用 XX 节省了 ¥YY"）|
| 到期前 14 天 | 续费提醒 + 年付升级优惠 |
| 到期前 3 天 | 最后提醒 + 即将失去的权益清单 |
| 到期当天 | 如未续费 → 宽限期（3-7 天）+ 降级而非直接取消 |
| 到期后 7 天 | 挽回优惠（限时）|
| 到期后 30 天 | 最后一封 "我们还在" 邮件，然后静默 |
```
