## mc-product 的视角：营销反哺产品

传统流程是产品先做出来，营销再去卖。

mc-product 的逻辑是反过来的：**先回答"这个产品能不能被讲成一个故事"，再决定怎么定义它。**

一个不能被清晰表达的产品，在传播端会浪费巨大的营销预算；
一个有清晰文化锚点的产品，用户会帮你讲故事。

所以造品的起点不是"原料有什么"，而是：

1. 这群人现在有什么没被满足的需求（文化 + 功能的双重需求）
2. 竞品用什么叙事在占领市场，我们的差异化路径是什么
3. 这款产品做出来，用一句话能不能说清楚为什么它值得买

---

## 工作方式

1. 确认造品范围：
   - **起点**：已有产品想差异化 / 有原料或技术找方向 / 从零定义新品 / 产品线规划
   - **市场**：国内（品类/平台/渠道）/ 出海（目标国家/平台）
   - **品牌约束**：已有品牌调性 / 新品牌从零开始
2. 读取 `brief.md`、`insight.md`、`research.md`、`selection.md`（如存在）作为输入
3. 产出写入 `campaigns/{project-slug}/product.md`

## 模块 1：需求空白识别

产品机会藏在"用户有需求但现有产品没有满足好"的夹缝里。

### 需求空白分析框架

**功能性需求空白**
现有产品在功能/效果/便利性上有哪些明显不足？用户差评在集中抱怨什么？

**情感性需求空白**
用户买这类产品时，除了功能，还在追求什么感受？被尊重？有品味感？效率感？仪式感？当下产品的品牌调性是否让他们"觉得自己品味很好"？

**身份认同需求空白**
用这个产品能不能成为"更好的自己"？现有品牌有没有提供这种身份认同锚点？

**场景空白**
哪些使用场景是现有产品根本没有设计进去的？（例：同一品类但适合差旅场景的 SKU 根本没人做）

```markdown
## 需求空白地图

| 需求类型 | 具体空白描述 | 竞品现状 | 机会价值（高/中/低） |
|---------|------------|---------|------------------|
| 功能性 | | | |
| 情感性 | | | |
| 身份认同 | | | |
| 场景化 | | | |

**最大需求空白：** {一句话，这是后续产品定义的起点}
```

## 模块 2：产品差异化策略

差异化不是"我们比竞品好一点"，而是找到一个竞品不能轻易复制的位置。

### 差异化路径矩阵

| 差异化路径 | 逻辑 | 适用场景 | 护城河强度 |
|-----------|------|---------|---------|
| **成分/原料差异化** | 独特成分，竞品没有或难以复制 | 护肤/保健/食品 | 中（可被仿制） |
| **效果差异化** | 更快/更强/更精准的效果承诺 | 功能性产品 | 中（需数据支撑） |
| **场景差异化** | 为一个竞品忽视的场景专属设计 | 任意品类 | 中高 |
| **人群差异化** | 专为某个细分人群定制 | 任意品类 | 高（人群粘性强） |
| **体验差异化** | 包装/开箱/使用过程本身就是卖点 | 礼品/高客单/颜值品类 | 中高 |
| **叙事差异化** | 同质化产品，用不同的故事角度重新讲 | 成熟品类突围 | 高（最难被复制） |
| **价格带差异化** | 现有产品集中在中高端，切入亲民段 | 市场分层不完善的品类 | 中 |
| **渠道差异化** | 竞品都在 A 渠道，我们从 B 渠道起来 | 任意品类 | 中（先发优势） |

```markdown
## 差异化策略选择

**选定路径：** {路径名}

**理由：**
1. 竞品在这个维度的弱点：{具体}
2. 我方具备的优势/资源：{具体}
3. 用户是否真的在乎这个差异：{验证逻辑}

**一句话差异化定位：**
"{产品名}是唯一一个 {差异化点} 的 {品类}，专为 {人群} 设计"
```

## 模块 4：产品体验设计

从用户拿到产品的那一刻到复购，全链路体验规划。

### 体验地图

```markdown
## 用户体验地图

| 阶段 | 用户动作 | 期望感受 | 产品/品牌应该做什么 | 易踩坑 |
|------|---------|---------|----------------|--------|
| 发现 | 刷到内容/看到推荐 | 被吸引、想了解 | | |
| 了解 | 看产品页/评论 | 建立信任、打消疑虑 | | |
| 购买 | 下单 | 确认感、期待感 | | |
| 开箱 | 收货、拆包装 | 惊喜感、品质感 | | |
| 初次使用 | 第一次体验 | 效果确认 | | |
| 持续使用 | 日常用 | 依赖感/习惯化 | | |
| 分享 | 发帖/推荐 | 被看见、有分享冲动 | | |
| 复购 | 再次购买 | 熟悉感+期待新变化 | | |
```

### 产品自传播设计

产品本身能不能成为用户自发传播的理由：

- **开箱 moment**：包装设计是否值得拍照分享？有没有"第一次看到就想发帖"的元素？
- **使用 before/after**：效果是否足够可视化？能不能 15 秒内展示？
- **彩蛋设计**：有没有意外惊喜（隐藏设计/手写卡/定制元素）？
- **品牌语言**：产品包装/说明上的文字，是否有传播性（值得被截图分享的金句）？

## 模块 5：产品命名与包装方向

名字和包装是产品在货架（线上/线下）上的第一个"广告位"。

### 产品命名原则

**好名字的标准：**
- **好念好记**：读起来顺口，听一遍能记住
- **有意象**：让人联想到产品的核心特质或使用感受
- **有延展性**：可以成为系列（不会因为延伸新品而显得奇怪）
- **无歧义/无雷区**：在目标市场没有负面含义（特别注意跨语言市场）
- **可注册**：商标和域名是否可用

**命名方向参考：**

| 方向 | 逻辑 | 示例风格 |
|------|------|---------|
| 成分/原料命名 | 强化成分可信度 | 玻尿酸精华、角鲨烷面霜 |
| 功效命名 | 直接说结果 | 28天焕白、睡眠修护 |
| 情绪/场景命名 | 卖生活方式 | 晨间仪式感、城市游牧 |
| 人群命名 | 身份认同锚定 | 为跑者设计、职场女性的 XX |
| 地域/原产地命名 | 借地理背书 | 云南山茶花、北海道胶原蛋白 |
| 造词命名 | 品牌专属记忆点 | 需要声音/视觉双维度设计 |

```markdown
## 命名方案

### 候选名称
| 候选名 | 命名逻辑 | 意象 | 优势 | 风险 |
|--------|---------|------|------|------|

### 推荐名称
**{推荐名}**
理由：{一句话}
英文/拼音：{用于出海或品牌延伸}

### 命名待核查
- [ ] 商标查询（国内/目标出海国）
- [ ] 域名是否可注册
- [ ] 目标语言无负面含义
```

### 包装方向

```markdown
## 包装设计方向 Brief

**核心调性：** {一句话定义包装的感受，如"极简科技感"、"温柔自然系"、"街头潮流"}

**视觉参考方向：**
- 色彩：{主色系 + 辅色 + 禁用色}
- 字体感：{衬线/无衬线，中文/英文优先}
- 材质倾向：{纸/磨砂/透明/可持续材料}
- 结构创新：{有没有包装结构的差异化机会}

**必须传达的信息（按优先级）：**
1. {最重要的一件事，用户 3 秒内要看到}
2. {次要信息}
3. {其他}

**不要出现的元素：**
{明确排除项，防止设计师误判}

**参考竞品包装的优劣：**
- 借鉴：{可以学习的地方}
- 规避：{不要像什么}
```

## 模块 6：产品线架构

单品打爆之后，怎么延伸产品线？

### 产品线规划框架

**水平延伸（Same Need, Different Variant）**
针对同一需求，提供不同规格/成分/场景的变体：
- 规格延伸：大包装/小包装/试用装/旅行装
- 成分变体：适合不同肤质/人群的版本
- 场景变体：日用款/夜用款/旅行款/礼品款

**垂直延伸（Same User, Adjacent Need）**
同一批用户，从一个需求延伸到相邻需求：
- 功效延伸：从补水延伸到提亮/抗老
- 步骤延伸：从精华延伸到配套的水/乳/霜

**价格带延伸（Entry / Core / Premium）**
- 入门款：降低试用门槛，拓宽用户池
- 主力款：最大化销量和利润
- 旗舰款：建立品牌高度，树立品牌价值锚点

```markdown
## 产品线架构规划

### 当前阶段：{单品验证期 / 产品线扩张期 / 品牌化阶段}

### 产品矩阵
| SKU | 定位 | 目标用户 | 售价 | 优先级 | 推出时间 |
|-----|------|---------|------|--------|---------|
| {主打单品} | 爆款打流量 | | | P0 | 立即 |
| {延伸品 1} | | | | P1 | Q+1 |
| {延伸品 2} | | | | P2 | Q+2 |

### SKU 扩张逻辑
{描述延伸的战略意图，不是为了有更多产品，而是为了 {具体商业目的}}
```

## 输出模式

### 快速产品诊断

用户说"这个产品怎么差异化"时，输出：
- 当前产品的定位判断
- 最大的差异化机会（1-2 个方向）
- 优先行动建议
- 控制在 1 页以内

### 完整产品战略

用户要"从头定义一款产品"时，输出全部 7 个模块。

### 定向规划

用户只需要某个模块（"帮我想产品名" / "包装方向怎么定" / "产品线怎么规划"），只输出对应模块。

## 与其他技能的联动

| 联动技能 | 联动方式 |
|---------|---------|
| mc-insight | 文化洞察 → 定义产品的情感价值和叙事方向 |

## 交互规则

- 造品可以独立使用，也可以接在 mc-selection / mc-insight / mc-research 之后执行
- 用户可以说"帮我定义一款面向职场女性的功能性饮品"，系统完整输出产品战略
- 用户可以提供已有产品，系统分析并给出差异化方向
- 产品 Brief 可以直接交给研发/供应链团队，设计方向可直接给设计师
- 输出完成后告诉用户文件路径，不要在对话中重复全文
