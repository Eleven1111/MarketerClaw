---
name: mc-dashboard
description: 营销健康度看板与异常预警技能。帮用户设计三层数据看板（高管/渠道/客户）、配置异常检测规则和预警阈值。用户说"帮我搭个看板"、"每天应该看什么数据"、"数据异常了怎么第一时间知道"时触发。产出可直接在 GA4/Looker/Tableau/飞书多维表格中落地的看板规格。
---

## 触发条件

当用户提到以下任意情形时使用本技能：

- "帮我搭个看板" / "数据看板" / "dashboard"
- "每天应该看什么数据" / "监控什么指标"
- "怎么第一时间发现问题" / "异常检测" / "预警"
- "健康检查" / "全盘数据" / "大盘怎么样"
- "日报/周报应该看什么"（侧重看板设计时触发，侧重报告内容时走 mc-report）

---

## 与其他技能的关系

| | mc-dashboard | mc-analytics | mc-diagnose |
|---|---|---|---|
| 时机 | 事前搭建（一次性设计 + 持续迭代） | 事中/事后（有数据时分析） | 出问题时（从症状找根因） |
| 产出 | 看板结构 + 异常规则 + 预警阈值 | 数据解读 + 决策建议 | 根因定位 + 修复路由 |
| 协作 | mc-dashboard 的预警触发 → mc-diagnose 深挖 → mc-analytics 跟踪修复效果 |

mc-dashboard 回答"应该盯什么数字、什么算异常"；mc-analytics 回答"这个数字什么意思"；mc-diagnose 回答"到底哪里出了问题"。

---

## 工作方式

1. 确认看板范围：
   - **业务类型**：电商 DTC / SaaS / 品牌种草 / 私域 / 综合
   - **当前阶段**：冷启动（月 GMV <10 万）/ 成长期 / 规模化
   - **工具环境**：GA4 / Looker / Tableau / 飞书多维表格 / Notion / 自建
2. 读取 `brand.md`、`strategy.md`（如有），对齐业务目标
3. 产出写入 `campaigns/{project-slug}/dashboard.md`

## 模块 2：异常检测体系

### 三层异常检测

```markdown
## 异常检测规则

### Layer 1：绝对阈值（硬规则）

超过即报警，不看趋势：

| 指标 | 红线 | 响应 |
|------|------|------|
| 网站 CVR | <0.5% | 立即排查技术问题 |
| 广告账户花费 | 单日 > 日预算 150% | 检查是否有异常放量 |
| 退货率 | 单日 >15% | 检查产品质量或物流问题 |
| 负面评价率 | 单日 >10% | 启动客服响应 |
| 页面 5xx 错误 | 任意出现 | 立即排查技术 |

### Layer 2：相对变化（环比检测）

| 指标 | 黄灯 | 红灯 | 对比基准 |
|------|------|------|---------|
| 日收入 | 环比 -20% | 环比 -40% | 同周同天 |
| 渠道 ROAS | 环比 -15% | 环比 -30% | 7 日均值 |
| 流量 | 环比 -25% | 环比 -50% | 7 日均值 |
| 加购率 | 环比 -20% | 环比 -35% | 7 日均值 |
| 邮件打开率 | 环比 -15% | 环比 -30% | 近 5 期均值 |

### Layer 3：动态基线（趋势检测）

适用于有 4 周以上历史数据的指标：

| 检测逻辑 | 说明 |
|---------|------|
| 移动平均偏离 | 当日值偏离 28 天移动平均 > 2 个标准差 |
| 连续方向性变化 | 连续 5 天/3 周同方向变化 |
| 季节性修正 | 对已知季节性（大促/节假日）自动调整基线 |

**动态基线启用条件：** 至少 4 周日粒度数据。数据不足时退回 Layer 1+2。
```

### IDA 诊断框架（异常触发后）

```markdown
## IDA 异常诊断流程

当异常检测触发后，按 IDA 三步定位根因：

### I — Identify（量化确认）
- 异常是真实的还是数据问题？
- 幅度多大？持续多久？
- 影响范围：全站 / 某渠道 / 某人群 / 某 SKU？

### D — Drill（交叉关联）
- 同时还有哪些指标在变？
- 相关渠道/人群/地域/设备的表现如何？
- 是否有已知的外部事件（大促/竞品动作/平台规则变化）？

### A — Act（维度钻取 → 行动）
- 定位到最小可操作单元（具体渠道 × 具体人群 × 具体内容）
- 给出修复建议 + 路由到对应技能
- 设定复查时间节点

**IDA 结论输出格式：**
| 步骤 | 结论 |
|------|------|
| I 确认 | {异常描述 + 幅度 + 范围} |
| D 关联 | {关联因素 + 排除因素} |
| A 行动 | {修复动作} → mc-{skill} |
```

## 模块 3：预警通知设计

```markdown
## 预警通知规格

### 通知分级

| 级别 | 条件 | 通知方式 | 响应时限 |
|------|------|---------|---------|
| 🔴 P0 紧急 | Layer 1 红线触发 | 即时推送（飞书/Slack/短信） | 1 小时内 |
| 🟡 P1 关注 | Layer 2 红灯触发 | 每日汇总推送 | 当天 |
| 🟢 P2 观察 | Layer 2 黄灯或 Layer 3 偏离 | 周报汇总 | 下次周会 |

### 通知内容模板

**P0 紧急通知：**
```
🚨 [{指标名}] 异常
当前值：{X}（红线：{Y}）
发生时间：{时间}
影响范围：{全站/渠道/人群}
建议动作：{一句话}
负责人：@{人名}
```

**P1 每日汇总：**
```
📊 {日期} 数据异常汇总
⚠️ {指标 1}：{当前值} vs 基准 {基准值}（{变化}%）
⚠️ {指标 2}：...
✅ 其余 {N} 个核心指标正常
```

### Alert Fatigue 防治

| 规则 | 说明 |
|------|------|
| P0 指标 ≤5 个 | 太多红线 = 没有红线 |
| 连续 3 天同一指标黄灯 → 自动升级为红灯 | 避免"一直黄但没人管" |
| 周末/节假日基线自动调整 | 避免周末流量低触发误报 |
| 同一指标 24 小时内不重复报警 | 避免刷屏 |
```

## 模块 4：看板落地指南

### 按工具出规格

```markdown
## 工具落地

### GA4 + Looker Studio（免费方案）
- 数据源：GA4 BigQuery Export
- 看板工具：Looker Studio（原 Google Data Studio）
- 优势：零成本，与 GA4 原生打通
- 限制：实时性取决于 BigQuery 导出频率

### 飞书多维表格（国内团队推荐）
- 数据源：API 对接 / 手动更新 / 飞书自动化
- 看板工具：飞书多维表格仪表盘
- 优势：协作强，通知与飞书打通
- 限制：需要手动或 API 更新数据

### Tableau / Power BI（企业级）
- 数据源：数据仓库直连
- 看板工具：Tableau Desktop + Server / Power BI
- 优势：可视化能力最强，复杂分析
- 限制：成本高，需要数据工程支持

### Notion（轻量级）
- 数据源：手动更新 / API
- 看板工具：Notion Database + 公式
- 优势：团队已在用，学习成本低
- 限制：数据分析能力弱，适合 <10 个指标
```

### 实施路线图

```markdown

|------|------|------|------|
```
