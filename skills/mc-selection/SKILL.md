- "选品" / "选什么产品卖" / "产品选品" / "爆款选品"
- "这个产品值不值得做" / "这个品类能不能切入"
- "帮我找几个热卖产品" / "有没有蓝海产品"
- "Amazon 选品" / "TikTok Shop 选品" / "独立站选什么卖"

- "竞争太激烈了换个品" / "找个差异化产品"

## 营销人选品视角

普通卖家选品看"能不能卖出去"，营销人选品看的是**"这个产品能不能被说故事"**：

- **传播性**：这个产品有没有让人想分享的理由？用户会自发发帖吗？
- **视觉化**：在图文/短视频里展示效果好不好？能不能做出吸引眼球的素材？
- **痛点清晰度**：用户的问题是否明确，解决方案是否可以一句话说清？
- **情感钩子**：有没有能打动用户的情绪价值（变美/变强/省事/有面子）？
- **口碑可复制性**：第一批买家会不会自发推荐？复购和裂变的逻辑是什么？

这五条比任何销量数据都先判断。一个在营销上无从着力的产品，即使现在好卖，也只是在打价格战。

## 工作方式

1. 确认选品场景：
   - **平台**：Amazon / 独立站 DTC / TikTok Shop / 小红书店铺 / 抖音小店 / Shopee / 国内天猫淘宝 / 多平台
   - **品类范围**：用户给出大方向（如"美妆"、"家居"、"3C 配件"）还是具体候选产品
   - **资源约束**：启动预算、供应链资源、团队能力
2. 读取 `brief.md`（如存在）了解品牌定位和目标人群
3. 产出写入 `campaigns/{project-slug}/selection.md`（如无 campaign 上下文，写入 `selection/{topic-slug}.md`）

## 模块 1：热度信号扫描

在判断产品潜力之前，先看市场信号是否真实存在。

### 需要扫描的信号源

**需求端信号（买家侧）**

| 信号源 | 看什么 | 判断逻辑 |
|--------|--------|---------|
| Google Trends | 关键词搜索趋势 12 个月 | 上升趋势 > 稳定 > 下降；季节性产品注意周期 |
| TikTok 话题播放量 | `#产品名` 的近 30 天涨幅 | 快速增长但未达亿级 = 爆款前期窗口 |
| 小红书笔记数量趋势 | 近 3 个月笔记量变化 | 从 0 到 1 的爆发期是最佳入场时机 |
| Amazon Best Sellers 排名变化 | Movers & Shakers 榜 | 排名快速上升 = 当前需求在爆发 |
| Pinterest Trends | 关键词 Pin 增长趋势 | 通常比 TikTok 提前 3-6 个月预示趋势 |
| 百度指数 / 微信指数 | 中国市场搜索热度 | 国内选品看这两个，而非 Google Trends |

**供给端信号（卖家侧）**

| 信号源 | 看什么 | 判断逻辑 |
|--------|--------|---------|
| AliExpress / 1688 | 月销量、下单趋势 | 工厂端出货量上升 = 下游零售端需求已经存在 |
| Amazon 同类产品评论数 | TOP 10 评论数量级 | 评论数 <200 = 蓝海；>5000 = 红海 |
| Shopify 店铺 Spy 工具 | 竞品独立站流量和畅销款 | 有成功案例但未规模化 = 机会窗口 |
| 淘宝/拼多多销量 | 同款产品月销量趋势 | 国内市场验证有效后出海机会大 |

```markdown
| 小红书笔记数 | {数量 + 近期增速} | | |
```

## 模块 2：需求真实性验证

热度高不等于真实购买需求。需要区分"话题热"和"钱包热"。

### 验证维度

**1. 搜索意图是否是购买意图**

区分信息型搜索和购买型搜索：
- "XX 是什么" → 信息型，离购买还远
- "XX 哪里买" / "XX 多少钱" / "最好的 XX" → 购买意图强

**2. 需求是否持续（vs 昙花一现）**

| 需求类型 | 特征 | 选品建议 |
|---------|------|---------|
| 常青需求 | Google Trends 5 年平稳，无明显季节性 | 低风险，慢热但长期有效 |
| 季节性需求 | 每年同期高峰，其余时间低谷 | 需要精准押注旺季时间窗口 |
| 趋势驱动需求 | 近 1-2 年快速上升 | 高机会，需判断是结构性增长还是短期风口 |
| 病毒性短期热度 | 某视频/事件带动，短期爆发后快速消退 | 高风险，不建议建立长期库存 |

**3. 支付意愿验证**

- 同类产品在 Amazon / Shopify 是否有定价 ≥ 利润线的成功案例？
- 用户评论里是否有"物超所值"或"就是有点贵但值得"这类表述？
- 有没有品牌溢价空间（同款白牌 vs 有品牌的价差）？

**4. 复购潜力**

| 复购类型 | 典型产品 | 商业价值 |
|---------|---------|---------|
| 消耗型（高复购） | 护肤品、保健品、咖啡 | LTV 高，适合订阅/会员模式 |
| 耐用型（低复购） | 家电、包包、家具 | 需要靠口碑和转介绍 |
| 升级型（周期复购） | 手机壳、运动装备、服装 | 靠季节更新或新款驱动 |

## 模块 3：竞争饱和度分析

市场够大才有意义，但竞争过度则无从突破。

### 饱和度判断矩阵

**Amazon 市场饱和度**

| 指标 | 蓝海 | 正常 | 红海 |
|------|------|------|------|
| 搜索结果第一页卖家数 | <5 个 BSR <1000 的 | 5-15 个 | >15 个且多大品牌 |
| TOP 10 平均评论数 | <200 | 200-2000 | >2000 |
| 最低广告 CPC | <$0.50 | $0.50-$2 | >$2 |
| 第一名评分 | <4.3 分（有改进空间） | 4.3-4.7 | >4.7（难超越） |
| 品牌集中度 | 多中小卖家 | 混合 | 1-2 个品牌垄断 |

**独立站/社交电商饱和度**

- TikTok 同话题视频：百万级以下 = 蓝海，亿级 = 红海
- 小红书同款笔记数：<1000 篇 = 早期，>10 万篇 = 饱和
- Google "site:shopify.com + 产品名" 搜索结果数量

### 差异化空间评估

即使市场竞争激烈，仍然值得进入的情况：

1. **用户痛点未被满足**：差评中高频出现某个问题，现有产品都没解决
2. **价格分层机会**：现有产品集中在中高端，中低端空白
3. **人群未覆盖**：产品好但营销针对错了人群，换一个细分人群重打
4. **内容差异化**：产品同质化，但内容方式有创新空间（新 POV、新场景）

```markdown
## 竞争饱和度分析

### 平台：{Amazon / 小红书 / TikTok Shop}
### 关键词：{搜索词}

| 饱和度指标 | 数据 | 判断 |
|-----------|------|------|
| | | |

**饱和度结论：** 🟢 蓝海 / 🟡 适度竞争 / 🔴 红海

**差异化突破口：**
1. {方向一}
```

## 模块 5：平台适配性评估

不同平台对产品有天然的筛选机制。选品必须与平台逻辑匹配。

### 各平台选品标准

**Amazon**

| 适合做 | 不适合做 |
|--------|---------|
| 标准化程度高、有明确功能 | 时尚/颜值导向（图片决定一切） |
| 搜索意图强（知道自己要买什么） | 需要内容教育才能理解价值 |
| 品类有搜索量 | 冲动消费型（线下/社媒场景更强） |
| 可通过 FBA 发货（重量 <10kg） | 危险品/液体/易碎品（FBA 限制多） |

**TikTok Shop / 抖音小店**

| 适合做 | 不适合做 |
|--------|---------|
| 视频展示效果强（before/after 明显） | 功能单一、展示无聊 |
| 能在 15-30 秒内说清价值 | 需要长篇文字说明 |
| 冲动消费属性（低决策门槛） | 高客单 / 强理性决策类产品 |
| 有情绪价值或娱乐价值 | 纯工具类、无情感共鸣 |
| 价格通常 <$50 | 高度标准化（价格战严重） |

**小红书店铺**

| 适合做 | 不适合做 |
|--------|---------|
| 颜值/生活方式相关 | 粗糙外观，包装简陋 |
| 女性消费者为主的品类 | 纯男性品类（用户基础薄弱） |
| 可以打出生活方式 POV | 无法与情感/身份认同挂钩 |
| 美妆/护肤/家居/服饰/宠物 | B2B 类、工业类 |
| 有 UGC 自传播潜力 | 无社交话题性 |

**独立站 DTC**

| 适合做 | 不适合做 |
|--------|---------|
| 可以讲品牌故事 | 纯价格竞争的白牌 |
| 有差异化定位和溢价空间 | 需要即时送达（竞争不过 Amazon Prime） |
| 有稳定复购（消耗品/订阅） | 一次性购买、无复购动力 |
| 适合内容营销驱动流量 | 需要大量实体展示才能决策 |

**Shopee / Lazada（东南亚）**

| 适合做 | 不适合做 |
|--------|---------|
| 价格敏感型产品（本地买家预算有限） | 高溢价品牌 |
| 小件轻货（跨境物流友好） | 大件重货（关税和物流成本高） |
| 东南亚市场有文化共鸣 | 欧美文化特有产品 |

## 模块 6：供应链可行性

好产品选出来，供应链跟不上等于零。

### 供应链评估维度

**货源渠道**

| 渠道 | 适用场景 | 优劣势 |
|------|---------|--------|
| 1688 / 阿里巴巴批发 | 小批量起步 | 起订量低，品控靠自己把关 |
| 工厂直采 | 规模化后 | 价格低，需要一定量，需验厂 |
| OEM/ODM | 想做差异化产品 | 可定制，MOQ 通常更高 |
| 海外本地采购 | 本地化产品，缩短物流 | 成本高，供应链更稳定 |
| 国内品牌出海 | 代理已有品牌 | 无需生产，但议价空间小 |

**关键供应链指标**

```markdown
## 供应链评估

| 维度 | 数据 | 判断 |
|------|------|------|
| 货源稳定性 | 供应商数量：{N} 家 | 单一/多元 |
| 最低起订量 (MOQ) | {数量} 件 | 是否在预算内 |
| 生产周期 (Lead Time) | {天数} 天 | 是否影响备货节奏 |
| 头程运费 | {金额/kg 或/件} | 是否影响利润模型 |
| 产品尺寸/重量 | {cm × cm × cm} / {g} | 是否触发超大超重费 |
| 品控难度 | 低/中/高 | 退货率风险 |
| 海关归类 | HS Code：{代码} | 关税税率是否在测算内 |
```

**物流模式选择**

| 物流模式 | 时效 | 成本 | 适用场景 |
|---------|------|------|---------|
| FBA（亚马逊入仓） | Prime 1-2 天 | 中等（有存储费） | Amazon 主战场 |
| FBM（自发货） | 7-15 天 | 低 | 测品期 / 大件产品 |
| 海外仓 | 3-7 天 | 中等 | 独立站 / 多平台 |
| 直邮（小包） | 10-20 天 | 低 | 低客单 / 测品 |
| 空运直发 | 5-10 天 | 高 | 高客单 / 旺季补货 |

## 模块 7：合规与风险筛查

这是大多数选品者忽视、但营销人必须提前排查的维度。

### 通用合规检查

```markdown
## 合规风险清单

| 风险项 | 状态 | 说明 |
|--------|------|------|
| 知识产权 / 专利 | 有风险 / 无风险 / 待查 | 是否有外观/功能专利壁垒 |
| 平台类目限制 | | Amazon/TikTok 是否有准入门槛 |
| 目标市场认证要求 | | CE / FCC / FDA / 3C 等 |
| 海关禁运/限制 | | 含磁/液体/动植物成分等 |
| 成分/材料合规 | | 食品/化妆品/儿童用品特别严格 |
| 广告合规 | | 功效宣称是否有法律风险（FTC/广告法） |
| 平台政策风险 | | 近期该品类是否有政策收紧信号 |
```

### 高风险品类预警

以下品类选品需要格外谨慎：

| 品类 | 风险类型 | 特别注意 |
|------|---------|---------|
| 保健品/食品 | FDA 认证、成分合规 | 功效宣称极易触发 FTC |
| 儿童用品 | CPSC、ASTM 安全标准 | 召回风险高，品控要求严格 |
| 电子产品 | FCC/CE 认证 | 无认证会被 Amazon 下架 |
| 美妆化妆品 | 成分备案（中国）、FDA（美国） | 含违禁成分直接翻车 |
| 宠物用品 | 安全性、材料毒性 | 用户对宠物安全高度敏感 |
| 仿牌/设计相似 | 知识产权 | 侵权封店是最常见的致命风险 |

## 模块 8：营销可行性评估

这是 mc-selection 区别于普通选品工具的核心模块。一个产品的营销可行性决定了它的天花板。

### 营销可行性评分维度

**1. 内容制作难度（越低越好）**

| 难度 | 描述 | 示例 |
|------|------|------|
| 低 | 效果立竿见影，素材易出 | 清洁产品污垢消除、健身器材使用前后 |
| 中 | 需要一定创意和拍摄技巧 | 服装穿搭展示、护肤品成分讲解 |
| 高 | 抽象价值，需要大量内容教育 | B2B 软件、保险、金融产品 |

**2. 用户自传播（UGC）潜力**

- 开箱体验是否有惊喜感？
- 使用效果是否可视化、值得拍照？
- 是否有话题性（适合发帖讨论）？
- 是否有情绪价值（让用户愿意晒出来）？

**3. KOL / KOC 适配性**

- 哪个层级的博主适合推这个产品？
- 腰部 KOC（1-10 万粉）能用吗？ROI 好不好？
- 专业博主推（专业背书）还是生活博主推（场景渗透）？
- 内容配合难度高不高（是否需要专业知识）？

**4. 钩子效应**

- 产品是否有"第一眼就想要"的视觉冲击力？
- 价格是否低到让人"反正先买一个试试"？
- 有没有赠品/套装/限时搭配可以提升感知价值？

### 营销可行性评分

```markdown
## 营销可行性评估
```

## 输出模式

### 快速筛选（默认）

用户说"这个产品值不值得做"时，输出：
- 市场信号（2-3 条核心数据点）
- 竞争判断（一句话）
- 利润粗估（售价 / 成本 / 利润率）
- 最大风险（一句话）
- **结论：推荐 / 有条件推荐 / 不建议**
- 控制在 1 页以内

### 完整选品报告

用户要"系统做一次选品"或"帮我对比几个候选品"时，输出全部 9 个模块。

### 定向分析

用户只关心某个维度（"这个产品 Amazon 竞争大吗" / "利润怎么算" / "供应链在哪找"），只输出对应模块。

## 交互规则

- 选品可以独立使用，也可以在 mc-campaign 启动前作为前置步骤
- 用户可以提供候选产品列表，系统逐一评估并对比排名
- 用户可以说"给我找几个热卖品推荐"，系统基于用户的平台和品类方向主动推荐
- 如果用户已有 brief.md，选品会自动聚焦到与品牌定位匹配的产品方向
- 选品结论可以直接作为 mc-campaign 的产品输入
- 输出完成后告诉用户文件路径，不要在对话中重复全文
