---
name: mc-review
description: 品牌调性 + 合规双审技能。支持中国广告法和海外法规（FTC/ASA/GDPR 等）。读取 campaign 产出或用户提供的文案，执行品牌一致性检查和法规风险扫描，输出通过/阻塞结论和具体修改项。
---

## 触发条件

当用户提到以下任意情形时使用本技能：

- "帮我审一下" / "品牌审校" / "合规检查" / "能不能上线"
- "这段文案有没有问题"
- "检查一下广告法风险" / "FTC compliant?" / "check compliance"
- 提供了一段文案或指向 campaign 目录中的文件

---

## 工作方式

1. 确定审查对象：
   - 如果用户指定了文件路径，读取该文件
   - 如果用户直接贴了文案，审查该文案
   - 如果用户说"审查整个 campaign"，读取 `campaigns/{project-slug}/` 下所有产出文件
2. 读取 `brief.md`（如存在）获取品牌调性、风险边界和表达禁区
3. **自动判断适用法规体系**：
   - 国内平台内容 → 中国广告法 + 平台规则
   - 海外平台内容 → 目标市场法规（FTC/ASA/EU 等）+ 平台规则
   - 跨境 campaign → 两套体系同时审
4. 将审查结果写入 `campaigns/{project-slug}/review.md`

---

## 品牌审校

### 检查维度

| 维度 | 具体检查项 |
|------|-----------|
| 调性一致 | 全部表达是否符合 brief 中的品牌调性定义 |
| 人群感知 | 目标人群看到后是否觉得"说的是我" |
| 卖点传达 | 核心卖点是否被充分支撑（用细节，不是宣称） |
| CTA 匹配 | 转化动作是否与品牌定调匹配（避免高端品牌用"立即抢购"） |
| 竞品区隔 | 表达是否有足够差异化，还是跟竞品撞脸 |
| 跨平台一致 | 不同平台的内容是否在调性上统一 |
| 本地化适配 | 海外内容是否有文化不适、直译痕迹或语感偏差 |

### 输出格式

```
### 品牌审校

**结论：** 通过 / 需修改

**发现：**
1. [通过/问题] 具体描述
2. ...

**修改建议：**（仅在"需修改"时输出）
1. 原文：{原始表达}
   建议改为：{修改后表达}
   原因：{为什么要改}
```

## 合规审查 — 法域规则按需加载

各法域的详细检查表存放在本技能目录的 `references/` 下，**只加载本次审查涉及的法域文件**，
不要全部加载：

| 适用法域 | 加载文件 |
|---------|---------|
| 中国广告法 + 国内平台规则 | `references/compliance-cn.md` |
| 美国 FTC + 美国平台规则 | `references/compliance-us.md` |
| 英国 ASA/CAP | `references/compliance-uk.md` |
| 欧盟 GDPR + EU 广告规则 | `references/compliance-eu.md` |
| 日本景品表示法 / 韩国表示广告法 | `references/compliance-jp-kr.md` |
| 东南亚各国 | `references/compliance-sea.md` |
| 敏感品类（食品/美妆/保健品/金融/教育/酒精/儿童，全球通用） | `references/sensitive-categories.md` |

加载规则：
- 法域由「工作方式」第 3 步的自动判断决定；跨境 campaign 加载国内 + 对应海外法域
- 内容涉及敏感品类时，**无论法域**追加加载 `sensitive-categories.md`
- `references/` 文件缺失时（不完整安装），按通用原则审查并在报告中标注
  「⚠️ 未加载 {法域} 详细规则表，结论需人工复核」

## 门禁判定

审查完成后，在文件顶部写入门禁状态：

```markdown
# 审查报告

**项目：** {项目名}
**审查时间：** {当前日期}
**审查范围：** {审查了哪些文件}
**适用法规体系：** {中国广告法 / FTC / ASA / EU / 多市场}

## 门禁状态

| 门禁 | 状态 | 说明 |
|------|------|------|
| 品牌审校 | ✅ 通过 / 🚫 需修改 | {一句话概括} |
| 合规审查（{市场}） | ✅ 通过 / 🚫 需修改 | {一句话概括} |
| **综合判定** | **✅ 可上线 / 🚫 未达上线标准** | |

> 所有生成内容仅供参考，上线前请由具备资质的法务或合规人员复核。
> 海外市场合规审查不构成法律意见，建议在目标市场聘请当地法律顾问确认。
```

## 交互规则

- 用户可以只做品牌审校（"只看品牌调性"）或只做合规审查（"只查合规"）
- 用户可以指定市场（"按美国 FTC 标准审"/ "按日本景品表示法审"）
- 跨境 campaign 默认同时审国内 + 海外法规，在报告中分章节呈现
- 用户修改文案后可以说"再审一遍"，系统重新审查并更新 review.md
- 审查粒度可调：用户可以说"只审 Instagram 的内容"
- 发现高风险问题时，在对话中直接高亮提醒，不只写在文件里

## 加权评分 rubric（合规硬门禁 + 质量软评分）

审校时按以下维度评分，**评分措辞本身就是质量转向器**——用"博物馆级文案 / 一眼记住 / 不可替代"
这类高标准锚点打分，逼出更高产出。

| 维度 | 类型 | 0-100 评分锚点 | 国内权重 | 海外权重 |
|------|------|---------------|---------|---------|
| 合规 | **硬门禁** | 通过 / 不通过（不打分） | 门禁 | 门禁 |
| 品牌一致性 | 软评分 | 100=与 brand.md 调性/人格高度统一；60=有偏移；<40=跑偏 | 0.3 | 0.4 |
| 原创性 | 软评分 | 100=自有判断与表达领土；60=有模板腔；<40=套话/撞脸竞品 | 0.3 | 0.3 |
| 转化杀伤力 | 软评分 | 100=钩子+卖点+CTA 三段咬合；60=平庸；<40=无转化驱动 | 0.4 | 0.3 |

判定规则：
- **合规不通过 → 该文件必出 `severity: block`**（不看质量分，触发反馈环重跑）。
- 任一**软评分维度 < 70 → 该文件出 `severity: warn`**（不阻断、不烧轮次，仅在交付卡提示）。
- 质量加权总分 = Σ(维度分 × 对应市场权重)，仅作参考写入 review.md，不单独门禁。
- 跨境 campaign 国内/海外内容分别用对应权重列。

## 结构化评审结论（仅 orchestrate 反馈环用）

在 mc-orchestrate 全链路中运行时，在常规审校输出**之后**追加机器可读尾块：

```
### 评审结论
verdict: pass | revise
scores:
  - file: content/小红书.md
    合规: pass | fail
    品牌一致性: 0-100
    原创性: 0-100
    转化杀伤力: 0-100
    加权总分: 0-100
findings:
  - file: content/小红书.md
    skill: mc-content
    severity: block | warn
    issue: "一句话问题描述"
```

规则：
- `合规: fail` 的文件**必须**有对应 `severity: block` finding；任一软维度 < 70 出 `severity: warn` finding。
- 只要存在 `severity: block` 的 finding，`verdict` 必须为 `revise`；全部为 warn 或无 finding 时 `verdict: pass`。
- `skill` 由上方反查表得出；`file` 用 campaign 内相对路径。
- **单步独立调用 mc-review（非 orchestrate 链路）时不输出此块**，保持现有终点式行为。
