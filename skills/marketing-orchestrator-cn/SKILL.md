## 触发条件

当用户提到以下任意情形时，使用本技能：
- 新品上市 / 品牌推广 / 节日大促 / 内容矩阵 / 复盘
- 需要"一套方案"、"完整 campaign"、"执行计划"
- 用自然语言描述项目背景、产品、人群、平台中任意一项

---

## 关键前提确认

若以下任何字段缺失，**先只追问缺失项，不做假设**：

| 字段 | 缺失时的影响 |
|------|-------------|
| 产品 / 项目名 | 所有交付物缺少主语 |
| 核心目标 | 策略无法聚焦 |
| 目标人群 | 内容角度和平台选择失准 |
| 主平台 | 无法输出平台专用资产 |
| 预算量级 | KPI 假设无法落地 |
| 风险边界 | 合规审查无参考标准 |

若用户说"直接给方案"，则对上述缺失字段做**合理假设并明确标注**，不停下询问。

## 标准工作流（按顺序执行）

### 1. 需求分诊台 — Brief 归一化
- 将用户输入整理为结构化 brief：项目名、产品、目标、人群、平台、预算、KPI、风险边界
- 指出信息缺口（若有），并给出默认假设
- 输出：`项目需求简报`

### 2. 策略负责人 — 传播策略
- 提炼核心信息屋：品牌主张 → 支撑理由 → 人群钩子
- 建立阶段节奏：破圈期/种草期/转化期/复盘期
- 确立差异化方向（对比竞品动作，而非泛泛叙述）
- 输出：`策略总纲`，结构为：定位 / 核心主张 / 阶段拆解 / 关键执行假设

### 3. 内容策划 — 内容矩阵
- 为主平台 + 次级平台分别输出内容方向
- 每个平台至少输出：
  - 2-3 个选题角度（含标题钩子样例）
  - 推荐内容形态（图文 / 短视频 / 直播 / 长文等）
  - 一条具体的开头句或脚本结构
- 语气严格对齐品牌调性，不输出空洞口号
- 输出：`内容矩阵`（平台 / 角度 / 格式 / 钩子 / CTA）

### 4. 渠道投放 — 渠道动作清单
- 主平台与次级平台的发布节奏（预热 / 引爆 / 收口）
- 达人资源建议（量级 / 频次 / 素材复用规则）
- 投放动作（资源位 / AB 测素材 / 优化信号定义）
- 约束处理：明确回应用户填写的渠道限制
- 输出：`渠道节奏表` + `执行 SOP 要点`

### 5. 品牌审校 — 品牌一致性核查
- 逐条检查策略、内容和渠道动作是否符合品牌调性
- 明确指出：不一致的表达 / 过度促销风险 / 人群感知偏差
- 对每条问题给出修改建议（不只是批评）
- 结论必须明确为：`通过` 或 `需修改`
- 输出：`品牌审校意见`（结论 / 发现 / 修改建议）

### 6. 合规审查 — 广告法与平台规则核查
- 逐条扫描：绝对化用语、功效承诺、医疗暗示、虚假优惠、平台敏感词
- 对每条风险标注等级（高 / 中 / 低）并给出替换文案
- 结论必须明确为：`通过` 或 `需修改`
- 输出：`合规审查意见`（结论 / 发现 / 风险等级 / 替换建议）

## 可选扩展（用户明确要求时执行）

若用户说"给可发的版本"、"直接给素材卡"或指定平台，追加输出：
- **小红书笔记卡**：标题（含关键词） / 正文（500字内） / 标签 / CTA
- **抖音脚本卡**：前3秒画面+口播 / 中段结构 / 结尾动作
- **微信长文框架**：开头设问 / 正文四段 / 私域承接动作
- **渠道一页纸**：面向执行团队或代理商，仅保留必要操作信息

若用户说"复盘"，将整体结构切换为：
目标回顾 → 数据表现 → 成功因子 → 问题归因 → 下次优化方向 → 可沉淀资产

---

## 输出原则

- 默认中文输出
- 每项建议必须可执行，不给"建议加强内容传播"这类空话
- 每个角色的结论必须有明确判断（通过 / 需修改 / 假设待确认）
- 若内容涉及健康、医疗、金融，额外给出"高风险提示"段落
- 最终结尾附上：**下一步行动清单**（按优先级列出 3-5 条）
