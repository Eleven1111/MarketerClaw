# mc-memory · 品牌记忆管理

## 触发条件

- "查看品牌记忆" / "品牌记忆是什么" / "品牌积累了什么"
- "记住这个洞察" / "保存这个发现" / "更新品牌记忆"
- "brand memory" / "memory" / "品牌积累"
- "切换品牌 {slug}" / "初始化品牌记忆"

---

## 文件路径规则

| 场景 | 路径 |
|------|------|
| 单品牌（默认） | `memory/brand-memory.md` |
| 多品牌 | `memory/{brand-slug}/brand-memory.md` |
| 品牌 slug 来源 | 当前 campaign 的 `brief.md` 中 `brand:` 字段；未指定时使用默认路径 |

---

## 操作清单

| 用户意图 | mc-memory 执行 |
|---------|---------------|
| 查看 | 读取 brand-memory.md，输出格式化摘要（不输出原始 markdown） |
| 手动添加洞察 | 将用户提供的内容追加到对应章节 |
| 手动修正 | 根据用户指令更新或删除某条记录 |
| 切换品牌 | 读取指定 brand-slug 的 brand-memory.md |
| 初始化 | 为新品牌创建空模板（`memory/{slug}/brand-memory.md`） |

---

## 写回规则

- 只追加，不覆盖已有内容
- `## Campaign 历史` 按 campaign slug 去重：写回前检查是否已有同一 slug 的行，有则跳过
- 内容明显重复（语义相同）的洞察跳过
- 文件不存在时自动创建空模板后写入
