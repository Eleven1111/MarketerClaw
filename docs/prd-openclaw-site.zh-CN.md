# MarketerClaw 产品需求文档（PRD · MVP）

## 1. 项目定位

MarketerClaw 是一套面向中国营销从业者的多角色协作工作流原型，运行在 OpenClaw 体系内，以 WebUI 作为操作界面直接调用后端 `/api/workflows/stream`。

**核心差异点：**
- 用户输入营销 brief，不是聊天 prompt
- 系统按角色（策略、内容、渠道、审核）自动推进工作流
- 内置中国营销语境的 Skills 包
- 输出策略摘要、内容资产、渠道动作、审核意见与复盘建议

## 2. 目标用户

- **Campaign 负责人**：需要一个完整营销计划的快速起点，用于对齐团队方向。
- **内容 / 渠道执行**：需要平台适配的可发布素材和渠道动作清单。
- **品牌 / 法务**：需要在交付前获得品牌调性和合规风险的结构化审查报告。

## 3. 核心场景（MVP）

1. 用户填写营销 brief（项目名称、产品、目标、人群、平台、预算、KPI 等）
2. 选择内置工作流模板（新品上市 / 大促 / 内容矩阵 / 复盘 / 自定义）
3. 配置每个营销角色绑定的模型（Mock / OpenAI / Google / 火山引擎）
4. 一键启动，系统以 SSE 流式返回各角色的实时执行进度和产出
5. 完成后查看：策略摘要、内容矩阵、平台资产卡、审核门禁结论、竞品洞察
6. 支持按指定角色或环节定向重跑，以及多格式导出（Markdown / JSON / PDF）

## 4. 功能范围（MVP）

### 前端（React + Vite + TypeScript）
- 营销 brief 配置表单
- 工作流模板选择与自定义编辑器
- 角色编制与模型配置
- 实时执行进度（SSE 流）与阶段看板
- 结果视图：策略、内容矩阵、平台资产卡、审核门禁、资产看板
- 历史运行列表与重跑
- 多格式导出入口

### 后端（Node.js + Express + TypeScript）
- `GET /api/health` 健康检查
- `GET /api/workflows/setup` 返回角色目录、模板、Skills 和平台
- `POST /api/workflows` 创建并同步执行工作流
- `POST /api/workflows/stream` SSE 流式执行
- `GET /api/workflows` 历史运行列表
- `GET/POST /api/workflows/:id/rerun` 全量重跑
- `POST /api/workflows/:id/rerun-from` 定向重跑（角色/环节）
- `GET /api/workflows/:id/export` 多格式导出

## 5. 角色体系

| 角色 | 分工 | 是否必选 |
|------|------|---------|
| 需求分诊台 | 整理 brief，补全缺失 | 必选 |
| 策略负责人 | 输出传播主张和阶段打法 | 必选 |
| 内容策划 | 内容矩阵、脚本、文案方向 | 必选 |
| 渠道投放 | 平台适配、节奏和执行动作 | 必选 |
| 品牌审校 | 调性一致性、禁用表达 | 必选 |
| 合规审查 | 广告法和平台规则风险 | 必选 |
| 数据分析 | KPI 假设、预算和监测 | 可选 |
| 营销自动化 | 链路配置和私域承接 | 可选 |
| 知识运营 | 模板沉淀和复盘卡 | 可选 |

## 6. 非功能要求

- 响应式 WebUI，支持主流桌面分辨率
- SSE 流式推送，无需页面刷新
- 模型 apiKey 不落盘（新版本已自动清除历史数据中的明文密钥）
- 工作流历史落盘到 `server/data/workflows`

## 7. 迭代建议

1. 迁移到正式的多 Agent 架构（OpenClaw Agent Registry 对接）
2. 支持真实 LLM 输出的 diff 预览与人工介入
3. 沉淀更多垂直场景模板（电商大促、品牌焕新、KOL 协同等）
4. 支持内容资产直接推送到代理商系统或 OKR 工具
